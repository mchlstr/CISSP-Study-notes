## Domain 1 — Security & Risk Management (glossary)

acceptable use policy (AUP) :: Policy defining permitted use of organizational systems and consequences for violations.
access control types :: Categories of controls (preventive, detective, corrective, deterrent, recovery, directive, compensating) implemented administratively, technically, or physically.
background checks :: Pre-hire investigation verifying a candidate's qualifications and disqualifiers.
baiting :: Social-engineering attack leaving infected media (e.g. USB drives) for victims to use.
bilateral NDA :: Mutual nondisclosure agreement where both parties protect each other's confidential info.
bottom-up approach :: Weak model where IT staff set security without senior-management direction.
business attack :: Attack aimed at stealing an organization's confidential data (corporate/industrial espionage).
California Consumer Privacy Act (CCPA) :: California privacy law modeled on the EU GDPR.
candidate screening :: Vetting an applicant per the sensitivity/classification of the role.
chief information security officer (CISO) :: Senior leader of information security reporting to executive management.
CIA Triad :: Core security goals of confidentiality, integrity, and availability.
administrative law :: Regulations issued by federal agencies, published in the Code of Federal Regulations.
administrative controls :: Policy/procedure-based controls (hiring, training, classification) enforcing security.
adverse actions :: Consequences imposed for violating company policy or committing offenses.
civil law :: Body of law governing disputes between parties, with monetary damages rather than imprisonment as remedy.
CLOUD Act :: 2018 U.S. law compelling U.S.-based providers to produce data under their control regardless of where it is stored.
clean-desk policy :: Administrative control requiring workers to secure sensitive materials when away from their workspace.
click-through license :: Software agreement accepted by clicking "I agree" during installation, binding the user to terms.
collusion :: Two or more people conspiring to bypass controls and commit fraud; mitigated by separation of duties.
computer-based training (CBT) :: Security/skills education delivered via computer rather than instructor-led.
Computer Fraud and Abuse Act :: Foundational U.S. anti-hacking law criminalizing unauthorized access to protected computers.
consensus (social proof) :: Social-engineering technique exploiting people's tendency to follow what others appear to do.
cost/benefit analysis :: Evaluating whether a safeguard's risk reduction justifies its cost.
cracker :: A malicious hacker who attacks systems with criminal intent.
criminal law :: Body of law prohibiting acts against society, enforced by the state with penalties like imprisonment.
criminal syndicates :: Organized-crime threat actors pursuing cybercrime for profit.
Cybersecurity Framework (CSF) :: NIST voluntary framework (Identify, Protect, Detect, Respond, Recover) for managing cyber risk.
cybersecurity insurance :: Insurance transferring financial risk of breaches, incidents, and cyberattacks.
DAD triad :: Disclosure, Alteration, Destruction; the failures opposing the CIA triad.
data localization :: Legal requirement to store/process certain data within a country's borders.
data minimization :: Collecting and retaining only the data necessary for the stated purpose.
data portability :: Individual's right to move their personal data between services/providers.
data sovereignty :: Principle that stored data is subject to the laws of the country where it physically resides.
Delphi technique :: Qualitative risk assessment reaching anonymous expert consensus through iterative rounds.
demonstrative evidence :: Illustrative evidence (charts, models) used to help explain testimony.
Digital Millennium Copyright Act (DMCA) :: U.S. law barring circumvention of copyright protections and giving ISPs safe-harbor.
direct evidence :: Evidence proving a fact through firsthand witness testimony without inference.
disinformation :: Deliberately false information spread to deceive or manipulate.
doxing :: Gathering and publicly releasing private information about a person/organization, usually to harm or intimidate.
due process :: Legal principle that government must respect a person's legal rights before depriving them of life, liberty, or property.
dumpster diving :: Recovering sensitive information by searching an organization's discarded trash.
Economic Espionage Act of 1996 :: US law criminalizing theft of trade secrets, with stiffer penalties when benefiting a foreign entity.
Electronic Communications Privacy Act (ECPA) :: US law criminalizing interception of electronic communications like email and voicemail.
eliciting information :: Social-engineering research technique of extracting information from people/systems to craft a pretext.
entrapment :: Law enforcement inducing someone to commit a crime they wouldn't otherwise have committed (a legal defense).
enticement :: Luring an already-willing offender (e.g., via honeypot) toward an act; legal, unlike entrapment.
espionage :: Covertly gathering confidential information to disclose or sell to a competitor or foreign government.
ethical disclosure :: Responsibly reporting a discovered vulnerability to the vendor to allow a fix before public release.
exit interview :: Debriefing of a departing employee (often to revoke access and gather feedback) during offboarding.
financial attack :: A crime aimed at unlawfully obtaining money or services.
Fourth Amendment :: US constitutional protection against unreasonable government search/seizure without a warrant and probable cause.
gamification :: Using game elements/rewards in security awareness to boost engagement and compliance.
grudge attack :: Attack motivated by resentment, often by a current/former employee, to damage an organization or person.
hacktivist :: Attacker who commits intrusions to advance a political/social cause, accepting consequences.
hoax :: Social-engineering trick (often a fake virus warning email) that scares users into harming their own systems.
identity theft :: Stealing and misusing another person's identifying information for fraud or impersonation.
ignoring risk :: Improper response of denying a risk exists; not a valid due-care strategy.
industrial espionage :: Illegally acquiring a competitor's confidential business information.
ISO 27002 :: Companion standard giving best-practice guidance for implementing ISO 27001 security controls.
ISO 27701 :: Extension of 27001 for privacy management (PIMS), aligning with regulations like GDPR.
issue-specific security policy :: Policy addressing a particular system, service, or function rather than the whole organization.
job rotation :: Rotating staff through roles to provide cross-training redundancy and detect/deter fraud.
management controls :: Administrative/policy-based controls that direct how security is governed and enforced organizationally.
managerial controls :: Controls focused on risk governance, planning, and oversight rather than technical implementation.
misinformation :: False or misleading information spread without malicious intent.
noncompete agreement :: Contract restricting an ex-employee from joining a competitor to protect trade secrets.
nonrepudiation :: Assurance a party cannot deny an action/message, e.g., via digital signatures.
operational plans :: Short-term, detailed plans derived from tactical plans, updated frequently.
organizational security policy :: Top-level policy addressing security across the whole organization.
parol evidence rule :: Legal rule that a written contract is presumed complete and unalterable by prior verbal terms.
plain view doctrine :: Legal rule letting an officer lawfully present seize evidence in plain sight tied to a crime, without a warrant.
prepending :: Adding text to a message header/subject to build pretext or evade filters in social engineering.
pretexting :: Social engineering using a fabricated identity/scenario to extract information from a victim.
privacy impact assessment (PIA) :: Analysis identifying privacy risks of a project and how to mitigate them.
privacy policy :: Internal-facing document governing how the organization handles employees' and others' PII.
POPIA :: South Africa's data-protection law (2020) regulating lawful processing of personal information, modeled on GDPR.
qualitative risk analysis :: Risk assessment using subjective rankings (high/medium/low) rather than dollar values.
real evidence :: Tangible physical objects admissible in court; aka object evidence.
realized risk :: A risk that has actually occurred as a breach, attack, or incident.
reasonable expectation of privacy :: Legal test of whether a person could reasonably assume they weren't being monitored.
reducing risk :: Implementing safeguards/countermeasures to lower vulnerability or threat impact; risk mitigation.
Regulatory policy :: Policy mandated by law/industry standards detailing required compliance procedures.
Rejecting risk :: Unacceptable response of denying or ignoring a risk; may constitute negligence.
Risk appetite :: Total amount of risk an organization is willing to accept in pursuit of its objectives.
Risk avoidance :: Choosing not to engage in an activity to eliminate its associated risk.
Risk capacity :: Maximum risk an organization can actually absorb, possibly less than its appetite.
Risk deterrence :: Implementing deterrents to discourage would-be violators.
Risk matrix :: Qualitative grid plotting probability against impact to rank risks (heat map).
Risk register :: Document inventorying identified risks for an organization, system, or project.
Risk tolerance :: How much realized-risk loss an organization is willing/able to absorb.
Role-based training :: Training tailored to employees' specific job duties and security responsibilities.
Script kiddie :: Unskilled attacker who runs prebuilt tools/scripts without understanding the underlying exploits.
Second-tier attack :: Follow-on attack that leverages data gathered from an earlier reconnaissance or eavesdropping attack.
Security framework :: Structured plan/model guiding the design and implementation of an organization's security program.
Security posture :: Overall state of an organization's security readiness and defenses.
semi-authorized hacker :: Gray-hat actor operating between ethical and malicious, often hacking without full authorization.
shoulder surfing :: Observing a user's screen or keystrokes to steal credentials or sensitive info.
skimming :: Covertly capturing card data via a planted reader on an ATM, POS, or gas pump.
smishing :: Social-engineering phishing conducted via SMS/text messages.
social engineering :: Manipulating people into divulging information or performing actions that compromise security.
spear phishing :: Targeted phishing crafted to appear from a known colleague or authority figure.
state actor :: Government/military-sponsored attacker (typically APT) targeting other nations.
strategic plan :: Long-term (about five-year) plan defining organizational mission, goals, and objectives.
succession planning :: Identifying and preparing personnel to fill key leadership roles when needed.
suicide hacker :: Highly destructive attacker unconcerned with being caught.
supply chain risk management (SCRM) :: Practices ensuring all vendors/links in the supply chain are trustworthy and disclose security practices.
system-specific security policy :: Policy targeting individual systems, prescribing approved hardware/software and required controls.
tactical plan :: Midterm (~1 year) plan detailing tasks needed to achieve strategic goals.
third-party governance :: External oversight of vendors/partners mandated by law, regulation, or contract.
threat actor :: The person or entity responsible for causing a security incident.
threat vector :: The path or method an attacker uses to reach and harm a target; aka attack vector.
transferring risk :: Shifting risk responsibility to another party via insurance or outsourcing.
trust but verify :: Traditional model trusting internal entities by default, leaving exposure to insider threats and lateral movement.
urgency :: Social-engineering pressure tactic exploiting fear of missing out to force quick action.
USA Patriot Act of 2001 :: Post-9/11 US law broadening surveillance and law-enforcement monitoring powers.

## Domain 2 — Asset Security (glossary)

asset owner :: Senior manager accountable for classifying and protecting an asset, delegating day-to-day care to a custodian.
big data :: Datasets too large for traditional processing, posing collection, storage, and analysis challenges.
cell suppression :: Hiding individual database cells to thwart inference and aggregation attacks.
cloud storage :: Data hosting on a provider's infrastructure, used for backup, sharing, or primary storage.
cryptographic erasure :: Sanitizing data by destroying its encryption keys so the still-present encrypted data can no longer be decrypted or accessed (the bits themselves are not actually erased).
data in transit :: Data moving across a network, protected by transport encryption.
data integrity :: Assurance that data is not altered between transmission and receipt.
data masking :: Obscuring sensitive data values while keeping it usable for non-production purposes.
data mining :: Analyzing large data warehouses to discover patterns and correlations.
data sanitization :: Removing data from media so it cannot be recovered.
data warehouse :: Large consolidated database aggregating data from many sources for analysis.
geotagging :: Embedding GPS location (and time) metadata in photos/messages, a privacy concern.
government/military classification :: Sensitivity labels: top secret, secret, confidential, unclassified (first three are "classified").
information life cycle :: Stages of data from creation through classification, storage, use, archival, and destruction.
provenance :: Verifying an item's origin and integrity by comparing it to its known original state/version.
pulping :: Paper destruction that shreds then dissolves documents into fibrous mush.
pulverizing :: Media destruction reducing devices to powder/grain.
reclassification :: Reevaluating an asset to assign a higher or lower classification as its value changes.
record retention :: Policy defining what records (often audit trails) to keep and for how long.
Removable media :: Portable storage media easily moved between systems, posing data-leakage risk.
system owner :: Role responsible for defining a system's requirements and ensuring its security.
USB data blocker :: Adapter that blocks USB data lines (allowing only power) to prevent juice-jacking.
whole-disk encryption (WDE) :: Encrypting an entire storage device under a master key; aka full-disk encryption.
wiping :: Overwriting/removing data from media to prevent recovery and data leakage.
zeroization :: Sanitization writing binary zeros to every bit of a storage device.

## Domain 3 — Security Architecture & Engineering (glossary)

access control triplet :: Clark-Wilson subject/program(transaction)/object relationship enforcing well-formed transactions.
analytic attack :: Cryptanalysis exploiting the algebraic/logical structure of a cipher to reduce its strength.
anything as a service (XaaS) :: Catch-all for any computing capability delivered via a cloud model.
application cells :: OS-level containers that virtualize applications for portability (application containers).
Address Space Layout Randomization (ASLR) :: OS defense randomizing memory layout to hinder exploitation.
authenticated cryptography mode :: Symmetric mode providing both confidentiality and integrity/authenticity (e.g. GCM, CCM, HMAC).
Bcrypt :: Key-stretching password hashing function based on Blowfish using salt and adaptive iterations.
Bell-LaPadula model :: Confidentiality-focused MAC/lattice model (no read up, no write down).
Biba model :: Integrity-focused MAC/lattice model (no read down, no write up).
blockchain :: Distributed, hash-linked ledger of records verified by timestamps and chained hashing.
boot attestation :: UEFI feature blocking unsigned OS/driver loading (secure boot).
boot security :: Protecting boot files/settings from malware via measured and secure boot.
Brewer and Nash model :: Model dynamically changing access based on prior activity to prevent conflicts of interest (ethical wall).
cascading :: Composition model where one system's output feeds another's input.
certificate chaining :: Linking root, intermediate, and leaf CAs to build a chain of trust to the end entity.
certificate path validation (CPV) :: Confirming every certificate from root to leaf in a chain is valid.
certificate policy :: Rules governing certificate use, behavior, and revocation conditions.
certificate practice statement (CPS) :: Document of the procedures a CA uses to issue and manage certificates.
chosen ciphertext attack :: Cryptanalysis where the attacker can decrypt selected ciphertext to break the cipher.
closed system :: Proprietary system designed to interoperate only with a narrow set of same-vendor products.
compartmented security mode :: System mode where all users hold clearance but need-to-know is enforced per compartment of data.
composition theories :: Information-flow security models (cascading, feedback, hookup) describing how system outputs feed others' inputs.
confinement :: Restricting a process to only read/write specified memory and resources (the Bell-LaPadula star property).
constrained data item (CDI) :: In Clark-Wilson, a data item whose integrity is protected by the model's rules.
containerization :: OS-level virtualization packaging an app with only its needed dependencies, sharing the host kernel without a full hypervisor.
container security :: Securing both the container host and the contents/configuration of individual containers.
covert storage channel :: Covert channel passing data by writing to a shared storage area read by another process.
cryptanalysis :: The study and practice of breaking ciphers and codes.
crypto module :: Hardware/software component providing crypto services like key storage, RNG, hashing, and encryption.
cryptographic key :: The secret value (cryptovariable) that parameterizes encryption and decryption.
cyber-physical system :: System bridging computation and the physical world via sensors and actuators (IoT/ICS).
data emanation :: Unintended electromagnetic signals leaking from equipment that can be eavesdropped (countered by TEMPEST).
digital envelope :: Secure key transport encrypting a symmetric key with the recipient's public key.
Digital Signature Standard (DSS/DSA) :: FIPS 186 signature method producing signatures via a dedicated algorithm rather than RSA encryption.
direct memory access (DMA) :: Mechanism letting devices exchange data with RAM directly, bypassing the CPU.
distributed system :: Multiple cooperating systems presenting as a single resource for resilience, scale, and performance.
diversity of defense :: Using different vendors/technologies at each layer so a single flaw doesn't compromise all controls.
downgrade attack :: Forcing parties to negotiate weaker crypto/protocol versions, often via an on-path attacker, to enable easier compromise.
dry pipe system :: Fire sprinkler holding pressurized air in the pipes; water only flows after the air bleeds out on activation, reducing accidental water damage.
dual-technology sensor :: Motion sensor combining two detection methods (e.g., IR + microwave) to reduce false alarms.
duress system :: A panic button/code that silently signals distress to a monitoring party for response.
edge computing :: Processing data on or near the device at the network edge to cut latency and bandwidth.
EEPROM :: Nonvolatile ROM that can be erased and rewritten electrically in place.
emission security (EMSEC) :: Controls (e.g., TEMPEST) preventing data leakage via compromising emanations.
entropy :: A measure of randomness/unpredictability, important for key and password strength.
ephemeral key :: A short-lived key generated for a single session/use then discarded, supporting forward secrecy.
EPROM :: PROM erasable by exposure to UV light through a window, then reprogrammable.
Escrowed Encryption Standard :: Failed US key-escrow scheme using the Clipper chip and Skipjack to give government backdoor access.
fault injection attack :: Inducing external faults (voltage, glitches) in a crypto device to undermine its integrity and extract secrets.
Federal Information Processing Standard 140 (FIPS-140) :: US standard defining security requirements for cryptographic modules.
fire suppression :: Systems/actions to extinguish a fire (water, gas, etc.) protecting people and assets.
fixed-temperature detection :: Fire detector/sprinkler that triggers when a set temperature melts its trigger element.
flame-actuated detection :: Fast fire detector triggered by sensing the infrared energy of flames; costly, for high-risk areas.
flash memory :: Nonvolatile EEPROM-derived storage erasable/writable in blocks; used in SSDs, USB drives, cards.
fog computing :: IIoT architecture where edge/IoT data is collected and processed at a nearby LAN node rather than far cloud.
geofencing :: Defining a geographic boundary (via GPS/Wi-Fi) to trigger mobile-device policies/features.
GNU Privacy Guard (GnuPG/GPG) :: Free open-source OpenPGP implementation for encryption and digital signatures.
Goguen-Meseguer model :: Integrity model based on predetermined domains of objects each subject may access (noninterference).
Graham-Denning model :: Access-control model defining secure operations for creating/deleting subjects and objects and assigning rights.
grid computing :: Parallel processing that loosely pools many distributed nodes toward a shared computation.
halon :: Ozone-depleting fire suppressant (now banned/replaced) that becomes toxic at high temperatures.
Harrison-Ruzzo-Ullman (HRU) model :: Access-control model (extends Graham-Denning) analyzing assignment and propagation of object rights to subjects.
hierarchical MAC environment :: MAC model where ordered classification levels grant access to a subject's level and all lower levels.
homomorphic encryption :: Scheme allowing computation on ciphertext so data stays encrypted while being processed (protects data in use).
hybrid cryptography :: Using asymmetric crypto to exchange a symmetric session key, then symmetric crypto for bulk data.
hybrid MAC environment :: MAC model combining hierarchical levels with compartments requiring both clearance and need-to-know.
in-band key exchange :: Exchanging keys within the same channel used for the protected communication.
Industrial Internet of Things (IIoT) :: IoT applied to industrial/infrastructure automation, integrating cloud data and analytics with ICS.
information flow model :: Security model controlling how information moves between objects/levels to prevent unauthorized flow.
IV attack :: Breaking encryption by exploiting weak, short, or predictable initialization vectors (e.g., WEP).
integrity verification procedure (IVP) :: Clark-Wilson procedure that confirms data items are in a valid, consistent state.
key recovery :: Retrieving stored encryption keys from backup or escrow.
key space :: Total range of possible valid keys for a given algorithm.
key stretching :: Techniques (PBKDF2, bcrypt) that strengthen weak keys/passwords against brute force.
Kerckhoffs' principle :: A cryptosystem's security should depend only on key secrecy, not on the algorithm being secret.
known plaintext attack :: Cryptanalysis where the attacker has matching plaintext-ciphertext pairs to help break the key.
known ciphertext attack :: Cryptanalysis using only captured ciphertext, exploiting key reuse or predictability.
lightweight cryptography :: Crypto designed for resource-constrained devices (IoT, smartcards, sensors).
link encryption :: Encrypting entire traffic (including headers) over each link, decrypting/re-encrypting at each hop.
Low Water-Mark Mandatory Access Control (LOMAC) :: Integrity-focused MAC implementation for Linux protecting processes and data.
M of N control :: Requiring a minimum M of N authorized agents to cooperate for a sensitive action (e.g., key recovery).
mathematical attack :: Cryptanalytic attack targeting weaknesses in the algorithm or key rather than brute force.
measured boot :: UEFI feature that hashes each boot component to attest boot integrity (vs. secure boot which blocks unsigned code).
meet-in-the-middle attack :: Cryptanalysis against double encryption that reduces effective key strength by matching encrypt/decrypt halves.
memory protection :: OS mechanism preventing a process from accessing memory not allocated to it (isolation).
message digest :: Fixed-length hash output summarizing a message's contents.
modes of operation :: Methods (ECB, CBC, CTR, GCM, etc.) for applying a block cipher across multiple blocks.
multilevel security mode :: System processing multiple classification levels at once where not all users are cleared for all data.
multiprocessing :: Using more than one physical processor to execute a single application.
multistate :: System certified to handle multiple security levels simultaneously with isolation mechanisms.
noninterference model :: Security model ensuring high-level subjects' actions don't affect what low-level subjects can observe.
nonvolatile storage :: Storage retaining data without power (disk, flash, NVRAM).
OCSP stapling :: Server attaches a signed, timestamped OCSP response to its certificate so clients needn't query the CA.
offline CA :: Root CA kept disconnected/powered off in secure storage, brought online only to sign subordinate certs.
one-way function :: Operation easy to compute forward but infeasible to reverse; basis of public-key crypto.
Opal :: Trusted Computing Group standard for self-encrypting drives using AES with pre-boot authentication.
OpenSSL :: Open-source library implementing TLS/SSL and cryptographic functions.
out-of-band key exchange :: Sharing an encryption key via a separate channel from the protected data.
POODLE :: SSL/TLS downgrade attack forcing fallback to vulnerable SSL 3.0.
pagefile :: Disk file holding memory pages swapped out under virtual memory; a forensic data source.
Password-Based Key Derivation Function 2 (PBKDF2) :: Key-stretching function applying salted hashing/HMAC over many iterations to slow password cracking.
PEM :: Base64-encoded certificate/key file format (.pem/.crt/.cer/.key).
pepper :: Secret value added to a password before hashing, stored separately from the database (unlike salt).
perfect forward secrecy :: Use of ephemeral session keys so compromising long-term keys doesn't expose past sessions.
PFX :: Binary certificate file format (.pfx/.p12) bundling certs and private keys, common on Windows; same as PKCS#12.
phlashing :: Malicious firmware/BIOS flashing that bricks a device or implants persistent malicious functionality.
physically unclonable function (PUF) :: Hardware feature generating a unique device fingerprint from inherent manufacturing variations.
PKCS#7 :: Cryptographic message/storage standard (used by S/MIME, often single sign-on); .p7b certificate container.
PKCS#12 :: Standard storing certs and private keys in one password-protected file container (PFX/.p12).
polyalphabetic substitution cipher :: Substitution cipher using multiple alphabets (e.g., Vigenere) to flatten frequency analysis.
post-quantum cryptography :: Algorithms based on problems believed hard even for quantum computers, to resist quantum attacks.
Pretty Good Privacy (PGP) :: Email security solution providing encryption and digital signatures via a web-of-trust key model.
Privacy Enhanced Mail (PEM) :: Email security standard providing authentication, integrity, confidentiality, and nonrepudiation using X.509.
private cloud :: Single-tenant cloud dedicated to one organization, on-prem or hosted, isolated from the public.
privileged mode :: CPU/kernel mode giving the OS the full instruction set; aka supervisor mode.
protected mode :: User mode with restricted hardware access and process isolation, distinct from kernel mode.
protection rings :: Architecture organizing OS/code into concentric privilege rings (ring 0 kernel to outer user rings).
public cloud :: Multitenant cloud service accessible to the general public over the internet.
public key cryptography :: Asymmetric scheme using a public/private key pair where one key encrypts and the other decrypts.
public ledger :: Distributed, replicated transaction log (blockchain) whose redundancy supports integrity.
quantum cryptography :: Using quantum properties of photons for cryptographic operations such as secure key exchange.
quantum key distribution (QKD) :: Quantum technique for two parties to establish a shared secret key, detecting eavesdropping.
RIPEMD-160 :: 160-bit hashing algorithm derived from RIPEMD/MD4, an SHA-1 alternative with limited adoption.
random number generator :: Source producing unpredictable values for cryptographic use.
ready state :: Process state where a process is prepared to run and awaiting CPU time.
real-time operating system (RTOS) :: OS designed to process data with deterministic, minimal latency for time-critical tasks.
renewal :: Reissuing a key or certificate with an extended validity period before expiration.
Restricted interface model :: Access model presenting different data/functions to subjects based on their classification level.
Restrictive defaults :: Configuring default settings to be limiting/secure (deny by default) to enhance security.
Rivest Cipher 5 (RC5) :: Fast block cipher with variable block/key sizes and rounds.
Root certificate :: Self-signed certificate a root CA issues to itself to establish the trust chain.
rooting :: Bypassing restrictions to gain full root privileges on an Android device (analogous to jailbreaking iOS).
Running key cipher :: Cipher whose key is a long agreed-upon text (e.g., a book passage) rather than a short fixed value.
SAN certificate :: X.509 cert with a Subject Alternative Name field covering multiple hostnames/IPs/identities.
Secure boot :: UEFI feature that only loads OS/drivers signed by trusted certificates, blocking boot-level malware.
Secure electronic transaction :: Legacy RSA/DES-based protocol for securing online credit card transactions, now abandoned.
Secure hash algorithm :: NIST hash family (SHA-1 160-bit; SHA-2 224/256/384/512; SHA-3) producing fixed-length digests.
Security boundary :: Line separating zones with differing security requirements or trust levels.
Security perimeter :: Boundary separating the trusted computing base from the rest of the system.
security through obscurity :: Flawed reliance on secrecy of design/implementation as the sole protection.
self-signed certificate :: Certificate signed by its own subject rather than a trusted CA; root CA certs are self-signed.
sensitivity label :: Classification marking attached to an object for access-control enforcement.
separation of privilege :: Granular per-operation permissions so privileged functions don't require full system access.
sideloading :: Installing apps onto a device outside the official app store, bypassing its vetting.
Signal Protocol :: Modern end-to-end encryption protocol for messaging, voice, and video.
silicon root of trust :: Tamper-resistant hardware anchor providing a trusted starting point for secure boot and integrity.
Simple Integrity Axiom :: Biba rule: a subject cannot read data at a lower integrity level ("no read-down").
single state :: System cleared to process only one classification level at a time, relying on policy to separate levels.
software-defined data center (SDDC) :: Virtualizing all data center infrastructure as software, often delivered as a service.
state machine model :: Security model ensuring a system remains secure in every possible state and transition.
statistical attack :: Cryptanalysis exploiting statistical weaknesses such as poor randomness in a cryptosystem.
storage segmentation :: Compartmentalizing data on a device, e.g., separating corporate from personal data on mobile.
substitution cipher :: Cipher replacing each plaintext element with a different one (mono- or polyalphabetic).
suspension :: Temporarily placing a certificate on hold so it can be reinstated before expiry.
swap file :: Disk file used as virtual memory to extend addressable RAM.
symmetric cryptography :: Encryption using one shared secret key for both encrypting and decrypting.
symmetric multiprocessing (SMP) :: Architecture where multiple CPUs share OS, memory, and bus, working jointly on tasks.
system call :: Mechanism by which a lower-privilege ring requests services/resources from a more trusted (kernel) ring.
system-high security mode :: MAC mode where all users are cleared for the highest data level but need-to-know is enforced per object.
Take-Grant model :: Access control model using a directed graph with take/grant/create/remove rules to govern rights transfer.
thin client :: Minimal-compute endpoint that relies on a server/VDI/mainframe for processing and storage.
transparency :: Quality of a security control being unseen by users so it can't be easily circumvented.
transposition cipher :: Cipher that rearranges (permutes) plaintext characters rather than substituting them.
trust anchor :: Inherently trusted, tamper-resistant element from which trust in a system is derived; the root of trust.
trusted operating system :: OS meeting security standards to enforce MAC with classification labels on subjects/objects.
trusted path :: Secure channel the TCB uses to communicate with users or the rest of the system.
type 1 hypervisor :: Bare-metal hypervisor installed directly on hardware with no host OS.
Type II hypervisor :: Hosted hypervisor running as an application atop a conventional host OS.
user mode :: Restricted CPU mode for executing non-privileged user applications.
Van Eck phreaking :: Eavesdropping by capturing and reconstructing electromagnetic emanations from devices.
VENONA :: US cryptanalysis project that broke a Soviet one-time-pad system due to pad reuse.
Vernam cipher :: Modulo-26 substitution implementing a one-time pad.
Vigenere cipher :: Polyalphabetic substitution cipher using a repeating keyword.
virtual machine :: Software-simulated computer with isolated memory running its own OS/process.
virtual private cloud (VPC) :: Isolated, customer-controlled section of a cloud provider's resources and virtual network.
volatile storage :: Memory like RAM that loses contents when power is removed.
weak key attack :: Cryptanalysis exploiting cipher keys with exploitable weaknesses.
wildcard certificate :: Certificate validating all subdomains of a registered domain.
work function :: Measure of a cryptosystem's strength by the cost/time needed to break it.
X.509 v3 :: Standard format for public-key certificates used by CAs.
zero-knowledge proof :: Protocol proving knowledge of a secret without revealing the secret itself.

## Domain 4 — Communication & Network Security (glossary)

802.1q :: IEEE standard for VLAN tagging of Ethernet frames across switches and trunks.
adversary-in-the-middle (AitM) attack :: Attacker intercepts and relays traffic between two parties, viewing/altering it (formerly MitM / on-path).
affinity :: Load-balancer setting that pins a client's requests to a specific backend server (session persistence).
air gap :: Physical isolation of a network/system with no connection to other networks.
always-on VPN :: VPN configured to auto-connect whenever a network link becomes active.
amplification attack :: DoS technique that multiplies attacker traffic (often via reflectors) to overwhelm a victim.
application-aware device :: Security device (firewall/IDS/IPS/proxy) inspecting content at upper protocol layers.
application firewall :: Filter enforcing strict rules for a specific application to block protocol/payload attacks.
Application layer :: OSI Layer 7, where applications access network services like file transfer and messaging.
application-level firewall :: Layer 7 firewall filtering traffic for a specific service, e.g. a web proxy (application gateway).
ARP cache poisoning :: Injecting forged IP-to-MAC mappings into the ARP cache to redirect traffic (ARP spoofing).
attenuation :: Weakening of a signal over distance, limiting cable run length and wireless range.
Automatic Private IP Addressing (APIPA) :: Self-assigned 169.254.x.y address used when DHCP fails (no gateway/DNS).
baseband :: Media supporting a single transmission/signal at a time.
beacon frame :: Wireless management frame broadcasting an AP's SSID and parameters.
Bridge Protocol Data Unit (BPDU) :: STP frame exchanged between switches carrying port status and identity.
broadcast storm :: Flood of unwanted broadcast traffic degrading network performance.
bus topology :: Network where all systems share a single backbone cable, risking collisions.

## Domain 1 — Security & Risk Management (recall terms)

What is an SLR? :: Service Level Requirement — what YOU (the customer) require and communicate to a vendor, before/forming the deal.
What is an SLA? :: Service Level Agreement — the formal CONTRACT committing the vendor to service levels (uptime, response), with penalties for failing.
What is an SLO? :: Service Level Objective — a specific measurable TARGET inside an SLA (e.g., 99.9% uptime).
What is an OLA? :: Operational Level Agreement — an INTERNAL agreement between your own teams that supports the SLA.
SLR vs SLA vs SLO vs OLA? :: SLR = requirement you state → SLA = the agreement (with penalties) → SLO = a target inside it → OLA = internal team support.
What is an ISA (interconnection)? :: Interconnection Security Agreement — a formal declaration of the security requirements for a network LINK between two organizations.
What is an MOU / MOA? :: Memorandum of Understanding/Agreement — an informal statement of INTENT; NOT a binding contract.
What is a BPA? :: Business Partner Agreement — defines the terms of a business partnership.
What is a VMS (vendor)? :: Vendor Management System — software to manage MANY vendors (consolidate billing, ordering, training).
Risk management frameworks — name them. :: NIST RMF, ISO 31000, COSO ERM, ISACA Risk IT, OCTAVE, FAIR, TARA. (CEH, CCMP, ARO are NOT frameworks — distractors.)
What is COSO ERM? :: Committee of Sponsoring Organizations — Enterprise Risk Management; enterprise risk + internal controls (financial/governance focus).
What is ISACA Risk IT? :: An IT-risk management framework (COBIT family).
What is OCTAVE? :: Operationally Critical Threat, Asset, and Vulnerability Evaluation — a self-directed risk-assessment framework.
What is FAIR? :: Factor Analysis of Information Risk — a QUANTITATIVE risk framework (puts dollar values on risk).
What is TARA? :: Threat Assessment and Remediation Analysis — a MITRE threat/risk framework.
The two axes a control is classified on? :: (1) TYPE/how implemented: Administrative (policy/people), Technical/Logical (tech), Physical. (2) FUNCTION/what it does: Deterrent, Preventive, Detective, Corrective, Recovery, Compensating, Directive.
Preventive control — mechanism? :: STOPS an action before it happens (lock, firewall rule, encryption, separation of duties).
Detective control — mechanism? :: IDENTIFIES/records an event after/while it happens (logs, IDS, CCTV review, audit).
Corrective control — mechanism? :: FIXES/returns to normal in place after an incident (patch, antivirus removing malware, IPS blocking the source). Full restore from backup = RECOVERY, not corrective.
Recovery control — mechanism vs corrective? :: Restores capability after a MAJOR event (DR site, backups, rebuild). Corrective = minor fix in place; recovery = larger-scale restoration.
Deterrent control — mechanism? :: DISCOURAGES a would-be actor (warning banner, "guard dog" sign, visible camera). Works on the mind, not the action.
Compensating control — mechanism? :: An ALTERNATIVE that provides similar protection when the primary control isn't feasible (e.g., can't patch → segment the host instead).
Directive control — mechanism? :: DIRECTS/mandates behavior (policy, procedure, signage telling people what to do).
A camera can be which functions, depending on framing? :: Deterrent (if visible, discourages) AND Detective (records for later). Same control, different function by use — read the stem.
If a question asks the FUNCTION of a control commonly labeled "physical," what do you answer? :: The function, not the type. Type and function are different axes — answer the one asked. A fence is physical by TYPE but preventive/deterrent by FUNCTION.
Asset / Threat / Vulnerability / Risk — mechanism of each? :: Asset = thing of value. Threat = potential harmful event/actor. Vulnerability = weakness. Risk = likelihood × impact that a threat exploits a vulnerability. Risk needs BOTH a threat AND a vulnerability.
Threat vs Threat agent/actor? :: Threat = the potential event (flood, malware). Threat agent/actor = the entity that carries it out (hacker, nature).
Exposure vs Risk? :: Exposure = an instance of being susceptible to loss. Risk = the probability+impact of that loss occurring.
Safeguard vs Countermeasure? :: Safeguard = PROACTIVE control (reduce risk before). Countermeasure = REACTIVE control (respond after a specific threat). Both are controls; timing differs.
Total / Inherent / Residual risk — how many real states? :: TWO. Total risk = Inherent risk = the SAME thing = risk BEFORE/without any controls. Residual risk = what's LEFT after controls (what you accept). Don't separate "total" from "inherent" — they're synonyms for "before controls."
Formula linking total and residual risk? :: Total (Inherent) risk − Controls gap = Residual risk. The controls gap = the amount of risk the safeguards removed.
Control gap — mechanism? :: The difference between total/inherent risk and residual risk — i.e., the amount of risk a control actually removes.
SLE / EF / ARO / ALE — formulas? :: SLE = AV × EF. ARO = times per year. ALE = SLE × ARO. EF = % of asset lost per incident. (n(n-1)/2 is the SYMMETRIC KEY formula, don't confuse.)
Qualitative vs Quantitative risk analysis — mechanism? :: Quantitative = hard numbers/money ($ALE). Qualitative = relative ratings (high/med/low, scenarios). Most real programs are hybrid.
Risk responses — the five? :: Mitigate/Reduce (add control), Transfer (insurance/outsource), Avoid (stop the activity), Accept (do nothing, document), Deter (discourage). You can never reach zero risk.
Due care vs Due diligence — mechanism? :: Due diligence = the ongoing investigation/research to KNOW (gather info, assess). Due care = DOING the reasonable actions (acting on it, implementing controls). Diligence = think/plan; Care = act. "A prudent person."
Policy / Standard / Baseline / Procedure / Guideline — mechanism + mandatory? :: Policy = high-level mandatory intent. Standard = mandatory specific requirement (must use AES-256). Baseline = mandatory minimum level. Procedure = mandatory step-by-step. Guideline = OPTIONAL recommendation. Only guideline is non-mandatory.
Data Controller vs Processor (GDPR)? :: Controller = decides WHY/HOW data is processed (the company). Processor = processes ON BEHALF of controller, per instructions (the vendor). Data Subject = the person the data is about.

## Domain 2 — Asset Security (recall terms)

Data Owner vs Custodian vs Steward? :: Owner = accountable, sets classification (senior/business). Custodian = implements/maintains protection day-to-day (IT). Steward = ensures data quality/business meaning. Owner decides, custodian does.
Clearing vs Purging vs Destruction? :: Clearing = overwrite, defeats normal recovery, media REUSED in same environment. Purging = stronger (degauss/secure-erase), defeats lab recovery, media leaves environment. Destruction = physically destroy, media gone. Escalating assurance.
Degaussing — what does and doesn't it work on? :: Works on MAGNETIC media (HDD, tape). Does NOTHING to SSD/flash (electrical charge, not magnetism) or optical. SSD → crypto-erase/secure-erase/destroy.
Tokenization / Masking / Anonymization / Pseudonymization? :: Tokenization = replace data with a token, real value stored in a vault (reversible via vault). Masking = hide part (****1234), often irreversible for display. Anonymization = strip identifiers permanently (NOT reversible). Pseudonymization = replace identifiers with a key that CAN re-identify (reversible, GDPR-relevant).
Steganography vs Watermarking vs DLP vs DRM? :: Steganography = HIDE data inside other data (covert). Watermark = embed identifying mark (trace leaks/ownership). DLP = detect/block sensitive data leaving. DRM = enforce usage rights on content.
Primary GOAL of asset management? :: Prevent losses (the outcome). Inventory/tagging/tracking are the ACTIVITIES that serve it. "Primary goal/purpose" → answer the business outcome, not the task.

## Domain 3 — Security Architecture & Engineering (recall terms)

What is CXL? :: Compute Express Link — a high-speed CPU-to-device interconnect. (Obscure, low-yield.)
Symmetric vs Asymmetric — mechanism + key math? :: Symmetric = one shared secret key, fast, bulk data; keys needed = n(n-1)/2. Asymmetric = public/private key pair, slow, key exchange + signatures; keys = 2n. Real systems use asymmetric to exchange a symmetric key (hybrid).
Confusion vs Diffusion? :: Confusion = relationship between key and ciphertext is complex (substitution). Diffusion = one plaintext bit changes many ciphertext bits (transposition/spreading). Confusion hides the key; diffusion spreads the plaintext.
Substitution vs Transposition? :: Substitution = REPLACE symbols (Caesar). Transposition = REORDER symbols (rearrange). Modern ciphers use both.
Hashing vs Encryption vs Encoding? :: Hashing = one-way, fixed-length, integrity (can't reverse). Encryption = reversible WITH a key, confidentiality. Encoding = reversible WITHOUT a key, just format (Base64) — NOT security.
Digital signature — mechanism + what it provides? :: Hash the message, encrypt the hash with SENDER'S PRIVATE key. Verify with sender's PUBLIC key. Provides integrity + authentication + non-repudiation (NOT confidentiality).
Non-repudiation — mechanism? :: Can't deny you did it — provided by digital signatures (private key only you hold). Symmetric/shared keys CAN'T give non-repudiation (others have the key).
Salt — mechanism? :: Random value added before hashing so identical inputs hash differently — defeats rainbow tables/precomputation. (Nonce/IV = randomness for encryption uniqueness.)
Which modes give confidentiality AND authenticity (AEAD)? :: GCM and CCM. ECB/CBC/CTR/OFB = confidentiality only. If a question wants integrity+confidentiality in one mode → GCM/CCM.
ECB weakness vs CBC error propagation? :: ECB = identical plaintext blocks → identical ciphertext (patterns leak), no chaining. CBC chains blocks, so a bit error PROPAGATES to the next block. CTR/OFB = stream-like, errors don't propagate.
Secure vs legacy symmetric ciphers (exam tactic)? :: Secure/modern: AES, Twofish, Serpent, Blowfish, CAST-256/CAST-128, RC6, IDEA. Legacy/weak (discontinue): DES, 3DES, RC4, SKIPJACK(80-bit). "Most secure" → pick the AES-family/finalist; "discontinue" → the old/short-key one.
AES facts (key sizes + max)? :: AES key sizes = 128/192/256 bits; MAX is 256 (there is NO 512). Block size = 128 bits always. One-time pad = the only THEORETICALLY unbreakable cipher (not AES).
Covert channel — storage vs timing? :: A path that transfers info in a way not intended. Storage channel = via a shared storage location; Timing channel = via modulating timing of events. Both bypass policy.

## Domain 4 — Communication & Network Security (recall terms)

What is SAE? :: Simultaneous Authentication of Equals — WPA3's replacement for pre-shared keys (Dragonfly handshake); resists offline dictionary attacks.
What is CCMP? :: Counter Mode CBC-MAC Protocol — WPA2's AES-based encryption.
OFDM vs FHSS vs DSSS — which is best and why? :: OFDM = highest throughput + LEAST interference (orthogonal subcarriers don't interfere). FHSS = hops between frequencies in a pattern; DSSS = spreads the signal across a wide band with a chipping code.
What is iSCSI? :: Internet Small Computer System Interface — SCSI storage commands over IP (SAN over standard Ethernet).
Store-and-forward vs cut-through switching? :: Store-and-forward = receives the WHOLE frame, checks for errors, then forwards (accuracy). Cut-through = forwards as soon as it reads the destination (speed, no error check). Accuracy > speed → store-and-forward.
What is a TCP wrapper? :: A host-based access control / basic firewall on Linux that restricts access by host or user ID.
What is SRTP? :: Secure Real-time Transport Protocol — encrypts VoIP/RTP media (protects voice calls from eavesdropping).
What is multicasting? :: One-to-many delivery to a group of subscribers (IGMP manages multicast group membership).
Traffic analysis — mechanism? :: Studies METADATA/patterns — who talks to whom, when, frequency, volume, size, timing — WITHOUT reading content. Works even on encrypted traffic. "Patterns and trends, not content" → traffic analysis.

## Domain 5 — Identity & Access Management (recall terms)

Identification / Authentication / Authorization / Accountability — mechanism? :: Identification = claim who you are (username). Authentication = prove it (password). Authorization = what you're allowed (permissions). Accountability = tie actions to identity via logging/audit. Order: Iden → Authn → Authz → Account.
Authentication factor Types 1/2/3 (+ extras)? :: Type 1 = something you KNOW (password). Type 2 = something you HAVE (token). Type 3 = something you ARE (biometric). Extras: somewhere you are (location), something you do (behavior). MFA = factors from DIFFERENT types.
FRR / FAR / CER — mechanism? :: FRR (Type I error) = False Reject, a legitimate user denied (safe but inconvenient). FAR (Type II error) = False Accept, an impostor let in (the dangerous one). CER = crossover where FRR=FAR; lower CER = better biometric.
Which biometric error is more dangerous and why? :: FAR / Type II (false accept) — it lets the wrong person in. FRR only inconveniences legitimate users.
DAC / MAC / RBAC / ABAC / Rule-based — mechanism? :: DAC = owner sets access at discretion (NTFS perms). MAC = system enforces via labels/clearances (military, mandatory). RBAC = access by job role. ABAC = access by attributes/policies (user+resource+context). Rule-based = global rules apply to all (firewall ACL).
Least privilege vs Need to know vs Separation of duties? :: Least privilege = minimum PERMISSIONS to do the job. Need to know = access only to DATA required. SoD = split a sensitive task across people so no one can complete fraud alone.
Keystroke MONITORING vs keystroke DYNAMICS (same root word, opposite mechanism)? :: Monitoring = keylogging, records the CONTENT of what's typed (surveillance). Dynamics = a BEHAVIORAL BIOMETRIC (Type 3, "something you do") measuring typing rhythm — dwell/flight time — to AUTHENTICATE the typist. Don't map "monitoring" to "dynamics."
Is keystroke dynamics the answer to "patterns and trends of DATA"? :: No. Keystroke dynamics = patterns of one person's TYPING (a biometric), not patterns of DATA FLOW. "Patterns/trends of data" = communication metadata = traffic analysis.

## Domain 6 — Security Assessment & Testing (recall terms)

What is SCAP? :: Security Content Automation Protocol — a suite of standards that automates vulnerability management.
SCAP: CVE vs CPE vs CVSS vs CCE? :: CVE = names VULNERABILITIES; CPE = names PLATFORMS (OS/apps/devices); CVSS = SEVERITY score 0–10; CCE = names CONFIGURATION issues.
What are synthetic transactions? :: Scripted, simulated user transactions run against an app to verify it behaves as expected (proactive monitoring/testing).
VA vs Penetration test? :: Vulnerability assessment = LISTS potential weaknesses (no exploit). Pen test = EXPLOITS to prove real impact. "Prove it could be reached" → pen test.
Blind / Double-blind / Black-white-gray box? :: Blind = tester has no info, defenders know. Double-blind = tester no info AND defenders not warned (tests detection/response). Black box = no internal knowledge; White box = full; Gray box = partial.
SAST vs DAST? :: SAST = static, reviews SOURCE CODE at rest (white-box, early). DAST = dynamic, tests the RUNNING app from outside (black-box).
Clipping level vs Sampling? :: Clipping level = a fixed THRESHOLD that drops events below it = NONstatistical data reduction. Sampling = select a representative subset = STATISTICAL. "Reduce audited data, nonstatistical" → clipping level.
Nmap / Nessus / Metasploit / lsof — primary job? :: Nmap = port scan/network map. Nessus = vulnerability scan. Metasploit = exploitation. lsof = list open files (local host diagnostic, not a scanner). Match the verb: "vuln scan" → Nessus; "port scan/map" → Nmap; "exploit" → Metasploit.

## Domain 7 — Security Operations (recall terms)

Electronic vaulting vs remote journaling vs remote mirroring? :: Vaulting = batch copies of backups sent off-site periodically; Journaling = transaction LOGS streamed off-site (smaller RPO); Mirroring = LIVE real-time copy at both sites (smallest RPO, most expensive).
What is clustering (for server failure)? :: Multiple servers act as one; if a server fails another takes over instantly (high availability / failover) — minimizes the impact of an imminent server failure.
What is UEM? :: Unified Endpoint Management — one tool to manage ALL endpoints (mobile + desktop + IoT); the evolution of MDM.
What is fileless malware? :: Malware that runs only in MEMORY, leaving no file on disk — evades file-based detection.
RTO / RPO / MTD / WRT — mechanism? :: RPO = max DATA loss tolerable (time before outage) → drives backup frequency. RTO = max time to RESTORE the system. WRT = time to verify/resume work after restore. MTD = total max downtime tolerable; RTO + WRT must be ≤ MTD.
MTBF vs MTTR? :: MTBF = mean time BETWEEN failures (reliability). MTTR = mean time TO repair. Don't confuse with RTO/RPO (those are tolerances; these are measured averages).
Hot / Warm / Cold site — mechanism? :: Hot = fully running, near-instant failover, most expensive. Warm = hardware+some data, hours to bring up. Cold = empty space/power, days, cheapest. Speed vs cost tradeoff.
Nondisaster / Disaster / Catastrophe? :: Nondisaster = service disruption, minor impact. Disaster = facility unusable for a day or more, short-term recovery. Catastrophe = facility destroyed, requires BOTH short- and long-term recovery. "Destroys the facility + long-term recovery" → catastrophe.
Fail-safe vs Fail-secure (and open/closed)? :: Fail-SAFE = fails to protect PEOPLE (doors unlock on fire = life safety). Fail-SECURE = fails to protect ASSETS (doors lock, data stays safe). "Fail-open" = available/unlocked; "fail-closed" = blocked/locked. Match to whether life or asset is prioritized.
Keystroke monitoring vs traffic analysis? :: Keystroke monitoring captures ACTUAL CONTENT (literal keys typed). Traffic analysis captures PATTERNS of flow, not content. They are opposites on the content axis.
Stem says "patterns/trends of data, NOT actual content" — which family of answer? :: The flow/metadata answer (traffic analysis). Eliminate anything that captures content/events: keystroke monitoring, event logging, security auditing all record content/events.

## Domain 8 — Software Development Security (recall terms)

What is SAMM, and its 5 business functions? :: Software Assurance Maturity Model (OWASP). Functions: Governance, Design, Implementation, Verification, Operations.
SW-CMM — the 5 stages in order? :: Initial → Repeatable → Defined → Managed → Optimizing.
What is a Fagan inspection? :: A formal, structured CODE REVIEW process (6 steps: planning → overview → preparation → inspection → rework → follow-up). Named after Michael Fagan.
Cardinality vs degree (database table)? :: Cardinality = number of ROWS; Degree = number of COLUMNS.
COTS vs library vs API vs open source? :: COTS = a finished commercial product; Library = reusable external code you pull INTO your app; API = an interface to interact with a service; open source = freely available source. Using external code IN your app = a Library.
Aggregation vs Inference — mechanism? :: Aggregation = combining individually-harmless pieces into sensitive whole (collecting access). Inference = DEDUCING sensitive info from clues you can see. Aggregation = collect; Inference = deduce.
Polyinstantiation — mechanism? :: Storing multiple versions of the same record at different classification levels to PREVENT inference (low-clearance user sees a cover story, not the secret).
TOCTOU / race condition — mechanism? :: Time-Of-Check to Time-Of-Use: state changes BETWEEN the check and the use, attacker exploits the gap. A timing/asynchronous attack.
Maintenance hook / backdoor — mechanism? :: Undocumented access left in code (often by developers) that bypasses normal authentication. Remove before production.
