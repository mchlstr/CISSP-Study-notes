# DRILL — Best-Answer Technique & Common Traps

These cards train a meta-skill rather than new facts:
when two answers are both "true," pick the BEST one.

## Goal vs Activity (the #1 trap)

Question asks "PRIMARY goal / purpose / objective" of a program — what level do you answer at? :: The business OUTCOME (the why), not the task (the how). Climb the ladder: inventory/patch/tag/scan are activities; "prevent losses / protect the org / support the mission" is the goal.
Asset management — primary goal: accurate inventory or loss prevention? :: Loss prevention. Inventory is the MEANS; loss prevention is the END. An accurate inventory is a true activity but not the goal.
Rule: when 3 answers are tasks/steps and 1 is an outcome/protection/business value, and the stem says "primary goal," pick? :: The outcome. Security programs exist to protect the organization / reduce loss / support the mission.
"What is the primary PURPOSE of X?" vs "Which is a COMPONENT/STEP of X?" — how does the right answer differ? :: Purpose → the high-level outcome (why). Component/step → the specific activity (how). Same facts, different abstraction level — read which one the stem wants.

## Primary tool / "primarily used to"

Watch-word "PRIMARILY" — what is it testing? :: Each tool/control is pinned to its CORE job even if it can do other things. Nmap can run vuln scripts, but PRIMARILY it's a port scanner; the vuln-scan answer is the dedicated scanner (Nessus).
"Network vulnerability scan" — Nmap or Nessus? :: Nessus. Nmap = port scan/mapping (primary). Real-world overlap doesn't beat the exam's primary-purpose mapping.

## Best-answer decision procedure (run this on every question)

5-step best-answer procedure when 2+ answers look correct? :: 1) Restate what's actually being ASKED (goal? first step? best? MOST?). 2) Eliminate clearly wrong. 3) For survivors, ask "which is MOST true / highest-level / addresses the ROOT?" 4) Match the stem's keyword (primary/first/best/MOST). 5) Pick the answer a manager/risk-owner would, not a technician.
When two answers are both technically true, the tie-breaker is usually? :: The one that is more COMPLETE, higher-level (managerial/risk), or addresses the ROOT CAUSE rather than a symptom. CISSP "thinks like a manager."
Stem says "FIRST" or "BEST first step" — bias toward? :: The earliest-in-the-process / foundational action (often assess, identify, get management support, determine scope) BEFORE technical execution.
Stem says "MOST effective / BEST / GREATEST" — bias toward? :: The control/answer with the broadest, root-level, preventive impact — not a narrow or after-the-fact fix.

## Exam mindset

Knowing the fact but picking the wrong option — what kind of error is that, and how is it fixed? :: A technique error, not a knowledge gap. Fix it by slowing down, restating the stem's keyword, and choosing the GOAL/ROOT/MANAGER-level answer rather than the first true statement.
Test anxiety after a hard question — what is the reset? :: Flag it, move on, refocus. Missing a two-true-answers item is normal and does not indicate a knowledge gap. Track the error pattern, not the emotional state.

## Qualifiers are the filter

A stem adds a qualifier ("nonstatistical," "without credentials," "at Layer 3," "automated," "primary," "least cost") — what is it for? :: It is NOT decoration — it's the discriminator. Two answers will both fit the verb; the qualifier is there to EXCLUDE one of them. Circle it and ask "which option does this rule out?"
Two answers both "reduce data" / "scan the network" / etc. — how do you split them? :: Find the qualifier word and apply it. Example: both clipping levels and sampling reduce log data, but "nonstatistical" excludes sampling (it's statistical) → clipping levels.
